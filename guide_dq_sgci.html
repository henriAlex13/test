À quoi sert ce framework ?

Chaque fois que tu extrais des données depuis Teradata dans Jupyter, tu as besoin de savoir si ces données sont fiables avant de les utiliser ou de les transmettre. Ce framework t'aide à répondre automatiquement à 6 questions essentielles.

Les 6 contrôles posés sur chaque extraction

01 Unicité
Y a-t-il des lignes entièrement dupliquées ?
Ex : la même transaction extraite 2 fois
02 Complétude
Le nombre de lignes est-il dans l'intervalle attendu ?
Ex : 0 lignes un jour de batch qui a planté
03 Fraîcheur
Les données contiennent-elles des lignes dans la période d'extraction ?
Ex : extraction de J-2 au lieu de J-1
04 Cohérence
Aucune colonne n'est-elle entièrement vide ?
Ex : BRANCH_CODE vide suite à une migration
05 Validité
Les colonnes sont-elles dans le bon type ?
Ex : MVT_ID retournée en float au lieu d'int
06 Exactitude
Les colonnes attendues sont-elles présentes ?
Ex : BRANCH_CODE et MVT_ID absentes de l'extraction

  • Si au moins une dimension est en ECHEC : aucun fichier n'est exporté. L'erreur est enregistrée dans le journal pour investigation.

Structure des dossiers

Après la première exécution, tu trouveras cette arborescence dans ton répertoire de travail :

le_projet/
├── mon_notebook.ipynb ← le notebook Jupyter
│
├── JOURNAL/
│ ├── sgci_dq.log ← log texte en temps réel
│ └── journal_dq.csv ← historique de toutes les exécutions
│
└── EXPORTS/
├── transactions_journalieres_20260615_060123.csv
└── ...

Le dossier JOURNAL

Le dossier JOURNAL contient deux fichiers complémentaires :

sgci_dq.log — log en temps réel avec horodatage. Il ressemble à :

LOG
2026-06-15 06:00:01 | INFO     | sgci_dq    | DQ — TRANSACTIONS JOURNALIERES | 250 lignes
2026-06-15 06:00:01 | INFO     | Unicité    | OK — Aucun doublon
2026-06-15 06:00:01 | INFO     | Complétude | OK — 250 lignes dans [50, 1000]
2026-06-15 06:00:02 | ERROR    | Cohérence  | ECHEC — Colonne(s) vide(s) : ['BRANCH_CODE']
2026-06-15 06:00:02 | WARNING  | Export annulé — 1 dimension(s) en ECHEC

journal_dq.csv — historique tabulaire de toutes les exécutions. Chaque ligne = une exécution. On peut l'ouvrir dans Excel ou avec pd.read_csv().

Le dossier EXPORTS

Contient uniquement les données ayant passé tous les contrôles. Les fichiers sont nommés automatiquement :

transactions_journalieres_20260615_060123.csv

Le logger

Le logger écrit automatiquement des messages horodatés dans deux endroits en simultané : le fichier sgci_dq.log (archivage) et la console Jupyter (temps réel). Il est utilisé en interne par run_dq(), pas besoin d'y toucher directement.

Étape 2 — Les 6 contrôles DQ

Les 6 fonctions de contrôle sont indépendantes. On n'a généralement pas besoin de les appeler directement — run_dq() le fait. Voici comment chacune fonctionne.

Contrôle 1 — Unicité

Python
check_unicite(df)

Utilise df.duplicated(keep=False) pour compter les lignes identiques sur toutes les colonnes. 2 lignes identiques colonne par colonne = doublons.

✅
"Aucun doublon" — toutes les lignes sont uniques
❌
"4 ligne(s) entièrement dupliquée(s)" — 2 paires de lignes identiques

Quand ça arrive : jointure SQL mal construite, extraction qui se chevauche, batch qui a tourné deux fois.

Contrôle 2 — Complétude

Python
check_completude(df, min_lignes=50, max_lignes=1000)

Compare len(df) à l'intervalle [min_lignes, max_lignes] que tu définis toi-même. Regarde les historiques des derniers jours pour calibrer l'intervalle avec une marge raisonnable.

✅
"250 ligne(s) dans [50, 1000]" — volume normal
❌
"0 ligne(s) — en dessous du minimum (50)" — table vide ou batch en échec

Contrôle 3 — Fraîcheur

Python
check_fraicheur(df, col_date="DATE", date_debut=hier, date_fin=hier)

Convertit les valeurs de col_date en dates Python, puis compte les lignes dans [date_debut, date_fin]. Si aucune ligne n'est dans cet intervalle, les données ne correspondent pas à la période attendue.

✅
"250 ligne(s) dans [2026-06-14, 2026-06-14]" — données fraîches
❌
"Aucune ligne dans [2026-06-14, 2026-06-14] — dates trouvées : 2026-06-13 → 2026-06-13"

Contrôle 4 — Cohérence

Python
check_coherence(df)

Parcourt toutes les colonnes et vérifie qu'aucune n'est entièrement nulle avec df[col].isnull().all(). Une colonne 100% vide signifie en général un problème SQL (mauvaise jointure, colonne inexistante).

✅
"Aucune colonne entièrement vide" — toutes les colonnes ont au moins une valeur
❌
"Colonne(s) entièrement vide(s) : ['BRANCH_CODE', 'REGION']"

Contrôle 5 — Validité

Python
check_validite(df, types_attendus={"AMOUNT": float, "TRN_REF": str, "MVT_ID": int})

Vérifie que le type pandas de chaque colonne correspond au type Python attendu. On ne liste que les colonnes importantes.

Type PythonSignificationExemple
strTexte"TRN00000001", "XOF"
floatNombre décimal125000.50, 0.0
intNombre entier42, 1000
boolBooléenTrue, False
✅
"Tous les types sont conformes"
❌
"'MVT_ID' : attendu numérique entier, trouvé float64"
Note : Teradata retourne souvent des types légèrement différents de ce qu'on attend. Ce contrôle détecte ces écarts avant qu'ils causent des erreurs plus loin dans le traitement.

Étape 3 — Lancer le pipeline avec run_dq()

run_dq() est la fonction centrale. Elle orchestre les 6 contrôles et décide si les données sont exportées ou non.

Signature complète

Python
rapport = run_dq(
    df             = df,                # DataFrame extrait de Teradata
    nom            = "mon_extraction",   # Nom descriptif (utilisé pour nommer le CSV)
    min_lignes     = 50,                # Nombre minimum de lignes attendu
    max_lignes     = 1000,              # Nombre maximum de lignes attendu
    col_date       = "MA_DATE",         # Colonne de date pour le contrôle fraîcheur
    date_debut     = hier,              # Début de la période attendue
    date_fin       = hier,              # Fin de la période attendue
    types_attendus = {                  # Types attendus pour les colonnes clés
        "MA_CLE"      : str,
        "MON_MONTANT" : float,
    }
)

Questions fréquentes

Puis-je utiliser ce framework pour des extractions hebdomadaires ou mensuelles ?

Oui. Il suffit d'adapter date_debut et date_fin. Exemple pour une extraction mensuelle de janvier 2026 :

date_debut = date(2026, 1, 1)
date_fin   = date(2026, 1, 31)
frequence  = "Mensuel"
Que faire si une dimension est en ECHEC ?

Commence par regarder le message d'erreur dans la console ou dans rapport["controles"]. Selon la dimension :

  • Unicité : vérifier si la requête SQL contient une jointure qui multiplie les lignes. Ajouter un DISTINCT ou revoir les conditions de jointure.
  • Complétude : vérifier si le batch source a bien tourné. Si 0 lignes, la table n'est peut-être pas encore alimentée.
  • Fraîcheur : vérifier le filtre de date dans la requête SQL. La table a peut-être un décalage de fuseau horaire.
  • Cohérence : la colonne vide existe-t-elle dans la table source ? Vérifier le nom exact et la jointure.
  • Validité : Teradata peut retourner des types inattendus. Convertir explicitement : df["MVT_ID"] = df["MVT_ID"].astype(int).
Peut-on avoir un export partiel (seulement les lignes valides) ?

Non, par choix de conception. Le framework exporte soit tout le DataFrame (si toutes les dimensions sont OK), soit rien. Cela garantit qu'un fichier présent dans EXPORTS/ est toujours 100% validé.

Si tu veux un export partiel (ex : exclure les doublons), déduplique d'abord le DataFrame avant de le passer à run_dq().

Comment ajouter un nouveau contrôle ?

Crée une fonction qui retourne {"statut": "OK"/"ECHEC", "message": "..."}, puis ajoute-la dans le dictionnaire controles dans run_dq() :

def check_mon_controle(df):
    # ... ta logique
    return {"statut": "OK", "message": "Tout va bien"}

# Dans run_dq() :
controles = {
    "Unicité"      : check_unicite(df),
    "Complétude"   : check_completude(df, min_lignes, max_lignes),
    "Fraîcheur"    : check_fraicheur(df, col_date, date_debut, date_fin),
    "Cohérence"    : check_coherence(df),
    "Validité"     : check_validite(df, types_attendus),
    "Mon contrôle" : check_mon_controle(df),  # ← ajout ici
}
Où trouver les fichiers exportés si j'oublie le chemin ?

Le chemin est toujours dans rapport["export_csv"] et dans la colonne Export_CSV du journal :

df_journal = pd.read_csv("./JOURNAL/journal_dq.csv", sep=";")
print(df_journal["Export_CSV"].dropna().tolist())
